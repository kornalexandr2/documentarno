# Documentarno: Руководство пользователя и Администратора

## О продукте
**Documentarno** — это Enterprise-grade система класса RAG (Retrieval-Augmented Generation) для работы с локальными документами в полностью изолированных (Air-Gapped) средах. 

Система автоматически отслеживает заданные папки (Watchdog), распознает сложные PDF-документы (включая таблицы) с помощью нейросетевого OCR, извлекает текст, векторизует его и помещает в высокопроизводительную базу данных. Встроенный ИИ-ассистент позволяет задавать вопросы по загруженным файлам, мгновенно находя нужные абзацы и предоставляя кликабельные ссылки на страницы оригинального документа во встроенном PDF-просмотре (Split-Screen RAG).

**Ключевые особенности:**
- **Абсолютная приватность:** Работает локально. Возможен запуск на серверах, физически отключенных от интернета.
- **Защита VRAM (Out-of-Memory):** Умная оркестрация моделей. Система автоматически выгружает LLM из памяти видеокарты перед запуском тяжелого OCR и наоборот.
- **Hardware Dashboard:** Мониторинг процессора, памяти, температуры GPU и дискового пространства в реальном времени с уведомлениями в Telegram.
- **Защита периметра:** Встроенный Rate Limiter (защита от брутфорса), теневое логирование чатов, ролевая модель и мгновенный аппаратный Lockdown (отсечение трафика на уровне Nginx) по одной кнопке.

## Системные требования

### Аппаратное обеспечение
* **Процессор (CPU):** От 4 физических ядер (рекомендуется 8+ ядер, например Intel Core i5/i7 последних поколений или AMD Ryzen 5/7).
* **Оперативная память (RAM):** 
  * **Минимум:** 16 ГБ (Возможны предупреждения о нехватке памяти).
  * **Рекомендуется:** 32 ГБ и более.
* **Видеокарта (GPU):**
  * **Рекомендуется:** NVIDIA GPU с объемом видеопамяти (VRAM) **от 8 ГБ** (например, RTX 3060, RTX 4060, RTX 4090, A-серия). Наличие GPU обеспечивает высокую скорость инференса LLM и распознавания текста.
  * **Без видеокарты (CPU Fallback):** Система *может* работать без GPU. При установке скрипт автоматически обнаружит отсутствие драйверов NVIDIA и переключит модели в режим CPU. **Внимание:** В этом режиме генерация ответов и распознавание PDF (OCR) будут работать **очень медленно**.
* **Дисковое пространство:**
  * **Тип:** Строго SSD или NVMe (HDD не рекомендуется из-за скорости работы БД).
  * **Объем:** От 50 ГБ свободного места для системы (БД, индексы Qdrant, Docker-образы) + место под скачанные ИИ-модели (от 5 до 20 ГБ каждая) + место для самих документов.

### Программное обеспечение (Хост-система)
* **ОС:** Linux (Ubuntu 20.04/22.04+, Debian 11/12+). Возможен запуск в LXC контейнере (Proxmox) или WSL2 (Windows 11).
* **Docker:** Установленный `docker` и плагин `docker-compose` (v2).
* **Драйверы GPU:** При наличии видеокарты NVIDIA должны быть установлены проприетарные драйверы и `nvidia-container-toolkit` для проброса GPU в Docker.

## Установка системы

1. **Подготовка репозитория:**
   Склонируйте или скопируйте папку проекта на ваш сервер.

2. **Запуск скрипта установки:**
   Перейдите в корневую папку проекта и выполните команду:
   ```bash
   chmod +x install.sh
   ./install.sh
   ```
   *Скрипт автоматически проверит наличие Docker, создаст нужные директории, скопирует файл `.env.example` в `.env`, проверит наличие видеокарты (и настроит Docker Compose соответствующим образом), соберет и запустит контейнеры.*

3. **Первичная настройка:**
   При первой установке вам будет предложено отредактировать файл `.env`. Обратите внимание на ключи:
   - `POSTGRES_PASSWORD` — пароль от БД.
   - `JWT_SECRET` — длинная случайная строка для криптографии токенов.
   - `FIRST_SUPERUSER` и `FIRST_SUPERUSER_PASSWORD` — логин и пароль для первого входа в систему (по умолчанию `admin` / `admin`). **Обязательно смените пароль после входа!**

4. **Доступ к системе:**
   После завершения работы `install.sh`, панель будет доступна по адресу `http://<IP-АДРЕС-СЕРВЕРА>:8080`.

## Варианты использования и Сетевая архитектура

Documentarno из коробки поднимает внутренний шлюз на базе Nginx, который слушает порт `8080` на хост-машине.

### 1. Локальная сеть (LAN)
Если сервер находится в закрытой корпоративной сети, дополнительная настройка не требуется. 
Просто открывайте в браузере `http://192.168.x.x:8080`.

### 2. Доступ через Интернет (Reverse Proxy)
Для публикации системы в интернет **категорически запрещено** просто открывать порт 8080 на роутере. Необходимо использовать внешний Reverse Proxy для обеспечения HTTPS (SSL/TLS шифрования) и дополнительной фильтрации.

Внешний прокси должен перенаправлять трафик на `http://127.0.0.1:8080` (или IP сервера Documentarno). Ниже представлены примеры настройки для популярных решений:

#### Nginx Proxy Manager (NPM)
NPM — самый простой вариант с графическим интерфейсом.
1. Добавьте новый Proxy Host.
2. Domain Names: `ai.yourcompany.com`
3. Scheme: `http`, Forward Hostname / IP: `IP_адрес_сервера_Documentarno`, Forward Port: `8080`.
4. Включите **Websockets Support** (обязательно для работы Hardware Dashboard и Streaming API).
5. На вкладке SSL запросите новый сертификат Let's Encrypt и включите `Force SSL`.

#### Традиционный Nginx (CLI)
Создайте файл `/etc/nginx/sites-available/documentarno.conf`:
```nginx
server {
    listen 80;
    server_name ai.yourcompany.com;
    return 301 https://$host$request_uri;
}

server {
    listen 443 ssl http2;
    server_name ai.yourcompany.com;

    ssl_certificate /etc/letsencrypt/live/ai.yourcompany.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/ai.yourcompany.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

#### Traefik
Если вы используете Traefik, добавьте лейблы в ваш `docker-compose.yml` к сервису `nginx`:
```yaml
  nginx:
    image: nginx:alpine
    # ... (существующие настройки) ...
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.doc-web.rule=Host(`ai.yourcompany.com`)"
      - "traefik.http.routers.doc-web.entrypoints=websecure"
      - "traefik.http.routers.doc-web.tls.certresolver=myresolver"
      - "traefik.http.services.doc-svc.loadbalancer.server.port=80"
```

## Работа с ИИ-моделями (Model Hub)

Для функционирования RAG-чата системе необходима языковая модель (LLM). Documentarno использует движок Ollama и формат моделей `.gguf`.

Управление моделями происходит в веб-интерфейсе, в разделе **Model Hub**. Возможны два сценария:

### Сценарий А: Сервер с доступом в Интернет
Если сервер имеет выход в сеть:
1. Перейдите на вкладку **"Скачать (Online)"**.
2. Введите тег модели из библиотеки Ollama (например, `llama3.1:8b` или `mixtral`).
3. Нажмите "Начать скачивание". Модель будет загружена напрямую в базу Ollama.

### Сценарий Б: Изолированный сервер (Air-Gapped)
Если сервер физически отключен от глобальной сети (для максимальной безопасности данных):
1. Скачайте нужную модель в формате `.gguf` на свой рабочий компьютер (например, с HuggingFace).
2. Откройте интерфейс Documentarno (через локальную сеть) и перейдите в **Model Hub** -> вкладка **"Загрузить (Offline)"**.
3. Укажите желаемое имя модели и выберите скачанный `.gguf` файл.
4. Дождитесь окончания загрузки. Система сохранит файл частями прямо в защищенный Docker Volume и автоматически зарегистрирует его в Ollama.

*Альтернативный путь для сисадминов:* Вы можете скопировать `.gguf` файлы по SSH или через флешку напрямую в примонтированную папку `data/ollama/models/` на хост-машине.

## Режимы синхронизации документов (Watchdog)
Все оригиналы PDF загружаются в папку `doc_source` (или через веб-интерфейс). Системная служба Watchdog следит за этой папкой. 

В разделе **Settings (Настройки)** вы можете выбрать поведение системы при **удалении** физического файла из папки:
1. **SYNC_AUTO:** Полное зеркалирование. Если PDF удален с диска, система автоматически удаляет его из базы данных и очищает векторы из Qdrant.
2. **SYNC_ADD_ONLY:** Режим архивариуса. Система реагирует только на добавление новых файлов. Удаление оригинала не удалит знания ИИ (база и векторы сохранятся).
3. **SYNC_PROMPT:** Строгий контроль. При удалении файла система ставит процесс на паузу и отправляет запрос Администратору в Telegram-бот для принятия решения.

## Telegram Бот и Уведомления
Для получения уведомлений об угрозах (попытки брутфорса, перегрев GPU, нехватка места на системном диске, риск OOM):
1. Создайте бота через `@BotFather` в Telegram и получите Token.
2. Узнайте свой Chat ID.
3. Введите Token и Chat ID в разделе **Settings** веб-интерфейса Documentarno и сохраните изменения. Бот перезапустится автоматически.
4. Вы можете отправить боту команду `/unlock`, чтобы снять режим аппаратного Lockdown, если вы включили его ранее из веб-интерфейса при угрозе.
